디버깅이란 프로그램에서 발생하는 문법 오류나 논리 오류를 찾고 바로잡는 과정

로그를 찍는 것 보다 디버깅을 활용해서 에러를 찾는 방법을 습관화 하자

자료형 범위에서 int의 범위에서 에러가 발생하는 경우가 있음
디버깅을 실행했을때 음수가 나오고, 에러가 발생한다면 범위를 넘겨서 이러한 에러가 발생하는 것 일 수 도있다.
그렇다면 long형을 사용해서 해결하면된다.
*코테에서 자료형은 처음부터 long형을 사용해서 작성하자!

디버깅을 사용해서 찾을 수 있는 오류
1. 변수 초기화 오류
    ex) 반복문을 통해 값을 넣어줘야하는데, 변수 초기화를 해주지 않아서 값이 이상해지는 부분을 발견할 수 있음
2. 반복문에서 인덱스 지정 오류
    ex) 변수의 범위를 100,000까지 줬는데 반복문에서는 10,000밖에 주지 않는 등 범위가 다를 경우의 인덱스 에러를 발견할 수 있다.
3. 잘못된 변수 사용 오류
4. 자료형 범위 오류
    ex) 이부분은 int대신 long형 사용해서 해결하기, 값을 사용했을때 음수가 나오면 안되는 알고리즘에 음수가 등장한다면 자료형을 바꿔보기
